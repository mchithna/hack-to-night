# Vercel Deployment Guide for HTN26

## Quick Start: Deploy in 5 Minutes

### Step 1: Sign Up to Vercel
- Go to https://vercel.com/signup
- Connect your GitHub account

### Step 2: Select Your Database Provider

**Recommended: Vercel Postgres** (Integrated, easiest)
- Click "Add Integration" in Vercel dashboard
- Search "Postgres"
- Follow the setup

**Alternative: Supabase** (Free tier, no credit card)
1. Sign up at https://supabase.com
2. Create new project
3. Copy connection string from Settings → Database

**Alternative: Railway** (Includes free credits)
1. Sign up at https://railway.app
2. New Project → Add PostgreSQL
3. Copy connection string

---

## Step 3: Configure Environment Variables

In your Vercel project dashboard:

1. Go to **Settings** → **Environment Variables**
2. Add the following variables:

```
DATABASE_URL=postgresql://user:password@host:5432/database
JWT_SECRET=your-strong-secret-here (use openssl rand -base64 32)
NODE_ENV=production
```

**If using Vercel Postgres:**
- Vercel auto-creates `POSTGRES_PRISMA_URL` and `POSTGRES_URL`
- Use `POSTGRES_URL` as `DATABASE_URL`

---

## Step 4: Update Your Code for Vercel

### Modify next.config.ts
Add this to ensure API routes work on Vercel:

```typescript
/** @type {import('next').NextConfig} */
const nextConfig = {
  // Vercel serverless functions need this
  api: {
    responseLimit: '8mb',
  },
};

export default nextConfig;
```

### Update Database Connection
If using connection pooling, modify your database connection:

```typescript
// lib/db.ts or similar
const connectionString = process.env.DATABASE_URL;

// For Vercel Postgres with pooling:
// const connectionString = process.env.POSTGRES_URL_NON_POOLING || process.env.DATABASE_URL;
```

---

## Step 5: Deploy

### Option A: Auto-Deploy from GitHub (Recommended)
1. Go to https://vercel.com/new
2. Select your GitHub repo
3. Vercel auto-detects Next.js
4. Configure environment variables (step 3)
5. Click **Deploy**

### Option B: Manual Deploy
```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod

# Follow prompts, paste your DATABASE_URL when asked
```

---

## Step 6: Run Database Migrations

After first deployment, set up your database schema:

### Option A: Using Vercel UI
1. Go to **Deployments**
2. Click the production deployment
3. Go to **Functions** tab
4. Check logs for any database errors

### Option B: SSH into Vercel Function (Advanced)
```bash
vercel env pull  # Download production env vars
# Then run migrations locally with prod DB connection
```

### Option C: Use a Setup API Route
Create `pages/api/setup.ts`:
```typescript
export default async function handler(req, res) {
  if (req.method !== 'POST') return res.status(405).end();
  
  // Run migrations/setup here
  // This is one-time only - DELETE after running
  
  res.json({ success: true });
}
```

Then call it once:
```bash
curl -X POST https://your-domain.vercel.app/api/setup
```

## Post-Deployment

### Set Custom Domain
1. In Vercel dashboard → **Domains**
2. Add your domain
3. Update DNS records (Vercel provides them)
4. SSL auto-generated by Vercel

### Monitor & Debug
```bash
# View logs
vercel logs --prod

# View environment variables
vercel env list

# Pull latest env vars to test locally
vercel env pull
```

### Redeploy
```bash
# Just push to main branch - auto-deploys
git push origin main

# Or manual deploy
vercel --prod
```

## Troubleshooting

**"DATABASE_URL not found"**
- Check Vercel Environment Variables are set
- Verify correct DATABASE_URL format
- Redeploy after adding env vars

**"Serverless Function Error"**
- Check Vercel function logs
- Ensure database connection pooling is enabled
- Test locally: `vercel env pull && npm run dev`

**"Cold starts too slow"**
- This is normal for serverless
- Use connection pooling (Prisma, pgBouncer)
- Consider upgrading to Vercel Pro for faster cold starts

**Can't connect to database**
- Verify DATABASE_URL in Vercel is correct
- Check database firewall allows Vercel IPs
- For Vercel Postgres: already configured ✓

## Database Backup (Important!)

For Vercel Postgres:
1. Vercel dashboard → Storage → Postgres
2. Click **Backups**
3. Automatic daily backups enabled by default

For Supabase:
1. Project Settings → Backups
2. Download SQL export

For Railway:
1. Database settings
2. Enable automatic backups
